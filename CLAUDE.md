## Project Overview

This is a **Docusaurus 3.9.2** static site generator project for creating documentation and blog websites. The project uses React 19.0.0 and requires Node.js >= 20.0.

## Essential Commands

### Development
```bash
npm start           # Start development server (hot reload enabled)
npm run serve       # Serve production build locally
npm run clear       # Clear Docusaurus cache (useful for troubleshooting)
```

### Build & Deploy
```bash
npm run build       # Generate static site in /build directory
npm run deploy      # Deploy to GitHub Pages (requires git remote setup)
```

### Content Management
```bash
npm run write-translations      # Generate translation files for i18n
npm run write-heading-ids       # Auto-generate heading IDs in markdown
```

## Architecture

### Configuration Flow
- **docusaurus.config.js**: Central configuration file
  - Site metadata (title, tagline, URL)
  - Theme configuration (navbar, footer, color mode)
  - Plugin and preset settings
  - i18n and deployment settings
  - Must update `url`, `organizationName`, `projectName` for deployment

### Content Structure
- **docs/**: Documentation pages (Markdown/MDX)
  - Uses autogenerated sidebar by default (configured in sidebars.js)
  - `intro.md` is the landing page for docs
  - Subdirectories create categories automatically

- **blog/**: Blog posts with frontmatter
  - File naming: `YYYY-MM-DD-title.md` or directory-based with `index.md`
  - `authors.yml`: Author metadata
  - `tags.yml`: Tag definitions

- **src/pages/**: Static pages (React components or Markdown)
  - `index.js`: Homepage (imports HomepageFeatures component)
  - Files here automatically become routes

### Component Architecture
- **src/components/**: Reusable React components
  - Use `@site/` alias for absolute imports
  - Example: `@site/src/components/HomepageFeatures`

- **src/css/custom.css**: Global styles and Infima CSS variable overrides
  - Define colors for light mode in `:root`
  - Define colors for dark mode in `[data-theme='dark']`

### Theme Imports
Docusaurus uses special import paths:
- `@docusaurus/Link`: Client-side routing link
- `@docusaurus/useDocusaurusContext`: Access site config
- `@theme/Layout`: Page layout wrapper
- `@theme/Heading`: Accessible heading component
- `@site/`: Alias for project root

### Sidebar Configuration
- **sidebars.js**: Controls doc navigation
  - Currently set to `autogenerated` from docs folder structure
  - Can be manually configured for custom ordering/grouping
  - `_category_.json` files in docs subdirs control category metadata

## Important Notes

### Before Deployment
1. Update `docusaurus.config.js`:
   - `url`: Your site's production URL
   - `organizationName`: GitHub username/org
   - `projectName`: Repository name
   - `baseUrl`: Usually `/` or `/<repo-name>/` for GitHub Pages

2. Configure git remote for deployment to work

### Dark Mode
The site respects system color preferences by default (`respectPrefersColorScheme: true`). Custom colors are defined using CSS variables in `src/css/custom.css`.

### Future Compatibility
The config includes `future.v4: true` for better compatibility with upcoming Docusaurus v4.

### MDX Support
Files with `.mdx` extension support JSX/React components within markdown. Use this for interactive documentation.
